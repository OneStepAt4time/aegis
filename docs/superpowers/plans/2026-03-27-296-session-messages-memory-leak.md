### Task 2: Add message cap to TranscriptViewer

**Files:**
- Modify: `dashboard/src/components/session/TranscriptViewer.tsx`

- [ ] **Step 1: Add MAX_SESSION_MESSAGES constant and cap logic**

In `dashboard/src/components/session/TranscriptViewer.tsx`:

Add near the top (after imports, before the component):
```ts
const MAX_SESSION_MESSAGES = 1000;
```

In the initial load `useEffect` (line 37-55), after `setMessages(msgs)`, cap the initial load:
```ts
getSessionMessages(sessionId)
  .then(data => {
    if (!cancelled) {
      const msgs = data.messages ?? [];
      const capped = msgs.length > MAX_SESSION_MESSAGES
        ? msgs.slice(msgs.length - MAX_SESSION_MESSAGES)
        : msgs;
      setMessages(capped);
      seenTimestamps.current = new Set(
        capped.map(m => m.timestamp).filter((t): t is string => !!t),
      );
    }
  })
```

In the SSE `useEffect` (line 66-69), cap after append:
```ts
setMessages(prev => {
  if (data.timestamp && seenTimestamps.current.has(data.timestamp)) return prev;
  if (data.timestamp) seenTimestamps.current.add(data.timestamp);
  const next = [...prev, data];
  return next.length > MAX_SESSION_MESSAGES
    ? next.slice(next.length - MAX_SESSION_MESSAGES)
    : next;
});
```
